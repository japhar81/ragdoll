# tone_profile_build

Builds a sender's tone profile from their outgoing messages. Two
outputs:

1. **Exemplars** — a curated subset of the sender's messages,
   filtered by length, deduped, and ranked by a simple "tone signal"
   score. Ready to wire into a `dataset_upsert` so a later retrieval
   pipeline (typically feeding `compose_with_style`) can pull the
   right few-shot examples per draft.
2. **Style guide** — a short prose description of the sender's voice
   (diction, sentence length, register, mannerisms) generated by ONE
   provider call over a representative sample.

Batch only — tone profiling is an offline preparation step, not a
hot-path operation.

## Inputs

- **`messages`** *(required)* — array of the sender's outgoing
  messages. Each must carry the configured `textField` (default
  `text`).

## Outputs

- **`exemplars`** — curated messages ready for a `dataset_upsert`.
- **`styleGuide`** — single-string style guide.
- **`sampleCount`** — count of curated exemplars (zero ⇒ provider
  was never called).

## Gotchas

- **The curator drops one-liners and mega-replies.** Tunable via
  `minChars` / `maxChars`; defaults are 60 / 800.
- **Defaults to Anthropic + Claude Sonnet** because the style guide
  shows up in every subsequent `compose_with_style` system prompt;
  it's worth doing once with a strong model.
- **One provider call per build**, regardless of sample size — the
  full curated set goes into the user prompt of that one call (up to
  20 samples).

## Typical position

Standalone "build tone profile" pipeline, invoked once per sender
during onboarding or whenever the user wants to refresh their voice:

```
mailbox_source → email_preprocess → tone_profile_build ─┬─ dataset_upsert(exemplars)
                                                        └─ secrets_set(style-guide secret)
```
